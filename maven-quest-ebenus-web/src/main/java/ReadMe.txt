Reponse question 1 :
Le serveur d'application est un middleware, il se situe entre la couche
os et l'application. Il founi les services recurrent aux applocations.
Il facilite donc le développement et le déploiement du code applicatif.




Reponse question 2 :
Tomcat n'est (en gros) qu'un conteneur de servlet.
GlassFish est un serveur d'application complet (conteneur servlet, ejb, etc...)


Reponse question 3 :
Les controllers sont les servlets
Les views sont les jsp
les models sont les utilisateursDao et roleDao
